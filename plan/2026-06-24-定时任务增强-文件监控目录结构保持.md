# 定时任务增强 + 文件监控目录结构保持

**日期**：2026-06-24

## Context

用户提出 3 个需求：
1. 定时任务新增"选择文件"按钮（目前只有"选择文件夹"）
2. 定时任务选择文件夹上传时，服务器端目录结构与本地一致
3. 文件监控任务文件被修改时也自动上传，且目录结构保持一致

### 2. scheduler.ts — 修复 buildRemotePath 多源路径支持

**文件:** `src/main/services/scheduler.ts`

当前 `buildRemotePath`（第 158-172 行）只用 `config.sourcePaths[0]` 作为 base。多源路径时，第二个源路径中的文件会 fallback 到 `path.basename`，丢失目录结构。

修复：遍历 `config.sourcePaths`，找到包含当前 `localPath` 的最长匹配源路径作为 base。

```ts
private buildRemotePath(localPath: string, config: ScheduleConfig): string {
  // 找到包含此文件的最长匹配源路径
  let baseSourcePath: string | undefined
  for (const sp of config.sourcePaths) {
    if (localPath.startsWith(sp) && (!baseSourcePath || sp.length > baseSourcePath.length)) {
      baseSourcePath = sp
    }
  }
  const relativePath = baseSourcePath
    ? path.relative(baseSourcePath, localPath)
    : path.basename(localPath)
  return path.join(config.remotePath, relativePath).replace(/\\/g, '/')
}
```

### 3. file-watcher.ts — 监听文件修改事件 + rename 为通用方法

**文件:** `src/main/services/file-watcher.ts`

当前 chokidar 只监听 `'add'`（第 39-41 行）。需新增 `'change'` 事件使文件修改时也触发上传。

改动：
- 将 `onFileAdded` 重命名为 `onFileEvent`（更通用）
- `addWatcher` 中新增 `change` 事件监听：

```ts
watcher.on('add', (filePath) => this.onFileEvent(filePath, config))
watcher.on('change', (filePath) => this.onFileEvent(filePath, config))
```

remotePath 构建逻辑 `path.relative(config.watchPath, filePath)` 已正确（第 140-141 行），无需修改。
