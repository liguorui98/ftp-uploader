# 下载进度弹框

为文件管理模块的下载功能增加实时进度弹框，显示下载进度、速度、剩余时间，支持取消下载。

## Context

当前 `browser:download` 是一个阻塞式 IPC 调用，下载期间 UI 无反馈。FTP/SFTP 客户端的 `download()` 方法虽然接受 `onProgress` 回调，但只在完成时触发一次。需要改为流式下载 + 实时进度推送 + 可取消。

| `src/main/services/ftp-client.ts` | FTPClient/SFTPClient download 改为流式下载，实时触发 onProgress |

## 步骤 1：FTP/SFTP 客户端 download 改为流式下载

**文件**: `src/main/services/ftp-client.ts`

### FTPClient.download

使用 Transform 流拦截数据块计算进度：先通过 `this.client.size(remotePath)` 获取文件大小，创建 PassThrough 流监听 `data` 事件累计字节并调用 `onProgress(transferred, total)`，再 `pipe` 到 WriteStream，最后调用 `this.client.downloadTo(passThrough, remotePath)`。

### SFTPClient.download

先通过 `this.client.stat(remotePath)` 获取文件大小，使用 `get` 传入 PassThrough 流拦截 chunk 计算进度。

## 步骤 2：IPC handler 增加进度推送和取消

**文件**: `src/main/ipc/index.ts`

### 修改 browser:download

- 调用 `client.download()` 传入 onProgress 回调
- 在 onProgress 中计算速度/ETA，通过 `mainWindow.webContents.send('browser:download-progress', data)` 推送
- 用 `cancelledDownloads: Set<string>` 管理取消状态
- 完成/失败/取消后发送 `browser:download-complete` 事件
- 返回 `{ success, localPath, downloadId }`

### 新增 browser:cancel-download

将 downloadId 加入 cancelledDownloads 集合。

### 速度/ETA 计算

复用 TransferManager 模式：`speed = (bytesDelta / timeDelta) * 1000`，`eta = remaining / speed * 1000`，300ms 限流。

## 步骤 5：ServerBrowser 进度弹框

**文件**: `src/renderer/pages/ServerBrowser.tsx`

- 新增 `downloading` / `downloadProgress` / `downloadId` 状态
- 下载流程：点击下载 → 获取 downloadId → 监听进度事件 → 显示 Modal
- Modal 内容：进度条 + 已下载/总大小 + 速度 + 剩余时间 + 取消按钮
- 监听 complete 事件自动关闭弹框并提示

## 验证

1. 点击文件下载 → 弹出进度弹框，实时显示进度、速度、剩余时间
2. 进度条平滑更新（300ms 间隔）
3. 下载完成弹框自动关闭，提示成功
4. 点击"取消下载" → 下载中断，弹框关闭，提示已取消
5. FTP 和 SFTP 服务器均正常
